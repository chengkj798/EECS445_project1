# Building Portfolio Lens: A Small Jac Project That Makes Portfolio Questions Easier

When I started learning Jac, I did not want my first project to be huge. I wanted something small enough to finish, but still useful enough that I would care about it. That is how I ended up building **Portfolio Lens**, a lightweight portfolio assistant that turns a simple list of holdings into plain-English answers, allocation views, and quick concentration checks.

The idea came from a simple frustration. A lot of portfolio tools show users too much at once. There are dashboards, metrics, watchlists, tables, and charts everywhere. That can be useful for advanced users, but sometimes I do not want a giant platform. Sometimes I just want to ask basic questions like: *What is my biggest position?* *Which sector takes the most weight?* *Am I too concentrated?* I wanted to build a version of that experience that felt simpler and more conversational.

So I kept the scope tight. Portfolio Lens takes in a small portfolio and lets the user ask natural-language questions about it. The app can return a short summary of the portfolio, identify the largest holdings, break the portfolio down by sector, and generate a basic allocation chart. The goal was not to create a full investing platform. The goal was to build a clean prototype that shows how Jac can support a more natural way to interact with structured financial data.

One reason this project felt like a good fit for Jac is that portfolios already have structure. A holding belongs to a sector. A sector contains multiple positions. A user question is usually asking for a path through that structure. Even in a small prototype, that made the project feel less like just another chatbot and more like a system with relationships. I liked the idea that the logic could follow the shape of the data instead of forcing everything into one flat workflow.

I built the demo around five public U.S. equities across four sectors. That kept the project realistic enough to feel grounded, but still small enough to explain clearly. The user enters holdings with prices, shares, and sector labels. From there, the backend organizes the data and routes the question to the right logic path. Some prompts are simple lookups, like finding the biggest position. Others need a small amount of aggregation, like calculating total sector exposure. I also added a chart view so the user could see the allocation instead of only reading text.

The hardest part was not making the chart or the holdings table. The hardest part was keeping the reasoning clear. A natural-language interface sounds easy at first, but even a basic finance question can mean different things. For example, a user asking what they are most exposed to might mean the biggest single holding, the largest sector, or a broader concentration pattern. That made me realize I had to design the project around clarity, not just around AI.

The biggest takeaway from this project is that good tools do not replace thinking. They reduce friction so I can spend more time on the actual problem. Jac helped me think in terms of structure, relationships, and clear logic paths. That was the part I found most valuable.

Portfolio Lens started as a small learning project, but it ended up teaching me something bigger. Building a useful prototype is not about adding as much as possible. It is about finding one meaningful interaction, making it work, and understanding why it works.
